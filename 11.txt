这个报错是因为 googletrans 库的 translate 方法是异步的（返回一个 coroutine 对象），但代码中没有使用
await 来等待它完成，导致试图访问 .text 属性时报错 'coroutine' object has no attribute 'text'。
修复说明：
我新增了 import asyncio，并将翻译函数拆分成异步版本(translate_text_async）和同步包装版本
(translate_text)，使用 asyncio.run()来运行异步调用。这在同步脚本中是可行的。
如果你在 PyCharm 中运行，确保 Python 版本支持 asyncio (3.7+ 都行，你的 3.13 没问题)。
对于 Mac（从路径看是 /Users/...），如果 Tesseract 路径有问题，可以在代码中指定：
pytesseract.pytesseract.tesseract_cmd = '/usr/local/bin/tesseract' (或你的实际路径,使用
which tesseract 命令检查)。
重新运行命令时,应该能正常翻译了。例如,如果提取的文字是 "Key stakeholders include:" (英文),选择 -
-action en_to_zh，应该输出中文翻译如“关键利益相关者包括：”。
